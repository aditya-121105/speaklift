# SpeakLift: Component System

To maintain visual consistency and development speed, components are strictly categorized into three tiers.

## 1. Foundation Components (The Core)
**Location**: `src/components/ui/`
**Source**: Primarily generated via `shadcn/ui` and Radix Primitives.

- **Definition**: The lowest-level atomic elements. Buttons, Inputs, Dialogs, Tooltips, Cards.
- **Responsibility**: Rendering UI states, managing accessibility (ARIA tags), and applying standard Tailwind classes.
- **Constraints**:
  - Must possess absolutely zero business logic.
  - Must never import anything from `src/features/`.
  - Must rely entirely on props to dictate state.
  - Must use `cva` (Class Variance Authority) for variant management.

## 2. Product Components (The Domain)
**Location**: `src/features/*/components/`

- **Definition**: Components built specifically for a SpeakLift feature, combining multiple Foundation Components. Examples: `InterviewFeedbackCard`, `SkillProgressChart`, `ResumeUploader`.
- **Responsibility**: Presenting domain-specific data structures.
- **Constraints**:
  - May receive complex domain objects as props (e.g., `AnswerEvaluation`).
  - Should avoid fetching data directly; prefer having data passed down from Experience Components.

## 3. Experience Components (The Orchestrators)
**Location**: `src/app/` (Pages) or top-level `src/features/*/components/`

- **Definition**: The top-level containers that orchestrate an entire flow or screen. Examples: `InterviewExecutionEngine`, `DashboardDashboard`, `ReportViewer`.
- **Responsibility**: 
  - Calling TanStack Query hooks to fetch data.
  - Managing Loading/Skeleton states.
  - Managing Error Boundaries.
  - Dispatching layout changes to Product Components.
- **Constraints**:
  - Should contain minimal actual markup; their job is to assemble Product Components.

## The AI Widget Framework
SpeakLift requires the AI to feel omnipresent. To achieve this without duplicating code, we use an architecture specifically for the AI identity.

**Location**: `src/components/ai/`

### 1. `AiAvatar`
- The visual anchor (the "Spark"). Handles the visual thinking state (pulsing gradient) vs. idle state.

### 2. `AiCoachMessage`
- A standardized wrapper for any text generated by the AI.
- Handles the typewriter rendering effect, syntax highlighting for technical answers, and markdown parsing.

### 3. `AiInsightCard`
- Used on the dashboard and inside reports. A specialized `Card` component that pairs the `AiAvatar` with a brief piece of actionable advice and an optional action button ("Practice Now").

By centralizing these components, if we decide to re-brand the AI's visual identity, we only update `src/components/ai/`, and the entire platform updates instantly.
