# SpeakLift: Emotional Design

Every screen in SpeakLift has a psychological objective. We are not just displaying data; we are managing the user's emotional state—often one of anxiety, imposter syndrome, or ambition.

## 1. Authentication (Login / Register)
- **Emotion**: Trust, Professionalism, Simplicity.
- **Design Execution**: A stark, beautiful, centered form. No distracting background illustrations. It must feel as secure and reliable as logging into a premium banking app.

## 2. The Dashboard
- **Emotion**: Control, Clarity, Motivation.
- **Design Execution**: The user must feel like they are stepping into the cockpit. The Readiness Score provides an immediate anchor of truth. The AI greeting makes them feel seen. The layout must be immaculate to prevent cognitive overload.

## 3. The Interview Session
- **Emotion**: Focus, Slight Pressure (Simulated).
- **Design Execution**: The UI strips away all navigation. The sidebar disappears. It is just the user, the camera/microphone indicator, and the AI prompt. The interface simulates the tunnel vision of a real interview. Distractions are fatal here.

## 4. The Interview Report
- **Emotion**: Relief, Curiosity, Constructive Acceptance.
- **Design Execution**: The first thing the user sees must be the overall score and the AI's *Positive* feedback. We buffer the critique. The colors are muted. The graphs are smooth. Even if the score is low, the UI must frame it as an exciting opportunity for rapid growth, not a failure.

## 5. Learning & Profile
- **Emotion**: Pride, Progression.
- **Design Execution**: This is the trophy room. Achievements, streak counters, and skill radar charts live here. It is highly visual and designed to make the user feel proud of the hours they have invested.

## 6. Error States
- **Emotion**: Forgiveness, Reassurance.
- **Design Execution**: Never use aggressive red alert boxes with technical jargon (`HTTP 500: UNEXPECTED_TOKEN`). Use conversational language ("Looks like we lost connection for a second. The AI saved your progress.") accompanied by a calming, abstract visual and a prominent "Retry" button.

## 7. Empty States
- **Emotion**: Invitation, Possibility.
- **Design Execution**: An empty dashboard should not look broken. It should look like a blank canvas waiting to be painted. Use inspiring micro-copy ("Your journey starts here") and clear calls to action.
