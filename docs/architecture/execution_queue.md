# Execution Queue Architecture (Materialized Path)

## Overview

SpeakLift's Adaptive Interview System (Milestone M2.2) introduces real-time generated follow-up questions. This creates a non-linear interview queue where questions are dynamically injected based on the candidate's performance.

To support infinite tree depth without collision, the system uses a **Materialized Path** string sorting architecture (`execution_path`) instead of a traditional integer sequence.

## Dual-Ordering Design

The system separates the concept of the *plan* from the *execution*:

### 1. Planned Order (`planned_order`: Integer)
- Generated by the `InterviewPlanner` before the interview begins.
- Strictly linear (1, 2, 3, 4, 5).
- Completely **immutable** during runtime.
- Used exclusively for reporting, analytics, and comparing actual interview execution against the original plan.

### 2. Execution Path (`execution_path`: String)
- Materialized path representing the actual dynamic interview sequence (e.g. `01`, `01.01`, `02`).
- Determines runtime presentation. All `ExecutionQuestionRepository` queues enforce `ORDER BY execution_path ASC`.
- Supports infinite fractional injections.

## Behavior

1. **Bootstrapping**: When the interview starts, primary questions inherit paths corresponding to their planned order, padded to two digits (e.g., `01`, `02`).
2. **Follow-Up Injection**: If a candidate answers poorly on `01`, the `AdaptiveDecisionEngine` triggers a follow-up. The `FollowUpGenerationService` derives the next child suffix by counting existing children of `01`. It appends `.01` to create `01.01`.
3. **Queue Sorting**: Because strings evaluate lexicographically, `01.01` is naturally sorted after `01` and before `02`.
4. **Infinite Depth**: If the engine generates a follow-up to `01.01`, it simply becomes `01.01.01`. The queue naturally shifts without complex reassignment loops or locking.

## Reporting Capability

The dual design allows queries to aggregate conversation branches.
To fetch all questions related to the first planned topic:
```sql
SELECT * FROM interview_questions 
WHERE planned_order = 1 
ORDER BY execution_path;
```

This ensures we never lose analytical traceability when generating dynamic tangents.
